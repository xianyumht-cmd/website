# 网站数据更新使用说明

## 概述

经过重构后，您现在可以通过修改配置文件来更新网站内容，而无需直接编辑HTML代码。这种方式更加安全、便捷，并且降低了出错的风险。

## 文件结构

```
fk/nav/
├── index.html          # 主页面文件（已重构，无需手动修改）
├── nav-data.js         # 数据配置文件（主要修改此文件）
└── 使用说明.md         # 本说明文件
```

## 如何更新网站内容

### 1. 更新游戏信息

打开 `nav-data.js` 文件，找到 `gameInfo` 部分：

```javascript
gameInfo: {
    version: "25.9",                    // 游戏版本号
    updateDate: "2025年4月30日",        // 更新日期
    nextUpdateDate: "2025年5月14日",    // 下次更新日期
    feedbackEmail: "admin@example.com"  // 反馈邮箱
}
```

### 2. 添加新的工具链接

在相应的分区中添加新项目。例如，在"常用工具"分区添加新工具：

```javascript
{
    title: "新工具名称",
    description: "工具描述",
    avatar: "图",  // 显示的单字符图标
    url: "https://example.com",
    tooltip: "鼠标悬停提示"
}
```

### 3. 修改现有链接

找到对应的项目，直接修改其属性：

```javascript
{
    title: "修改后的标题",
    description: "修改后的描述",
    avatar: "新",
    url: "https://new-url.com",
    tooltip: "新的提示信息"
}
```

### 4. 删除不需要的链接

直接删除对应的对象即可，注意保持JSON格式正确（逗号的使用）。

### 5. 添加新分区

在 `sections` 数组中添加新的分区：

```javascript
{
    id: "新分区ID",
    title: "新分区标题",
    icon: "fa fa-icon-name",  // FontAwesome图标类名
    items: [
        // 在这里添加该分区的工具项目
    ]
}
```

## 重要注意事项

### ⚠️ JSON格式要求

1. **逗号使用**：每个对象和数组项后面都需要逗号，但最后一项不能有逗号
2. **引号使用**：所有字符串必须用双引号包围
3. **括号匹配**：确保所有的 `{}`、`[]` 都正确匹配

### ✅ 修改步骤

1. 备份 `nav-data.js` 文件
2. 使用文本编辑器打开 `nav-data.js`
3. 按需修改内容
4. 保存文件
5. 刷新浏览器页面查看效果

### 🔧 常见问题排查

**页面显示空白或错误**：
- 检查 `nav-data.js` 的JSON格式是否正确
- 使用在线JSON验证工具检查语法
- 查看浏览器控制台的错误信息

**新添加的链接不显示**：
- 确认添加在正确的分区中
- 检查对象格式是否完整
- 确认没有语法错误

**图标不显示**：
- 确认使用的是有效的FontAwesome图标类名
- 检查图标类名拼写是否正确

## 优势对比

### 之前的方式（直接修改HTML）
- ❌ 容易破坏页面结构
- ❌ 代码混乱，难以维护
- ❌ 容易出现语法错误
- ❌ 需要了解HTML知识

### 现在的方式（修改配置文件）
- ✅ 数据与展示分离，更安全
- ✅ 结构清晰，易于理解
- ✅ 降低出错风险
- ✅ 无需HTML知识
- ✅ 便于批量修改和管理

## 技术说明

重构后的系统采用了数据驱动的方式：

1. **数据层**：`nav-data.js` 存储所有配置数据
2. **展示层**：`index.html` 通过JavaScript动态生成页面内容
3. **样式层**：保持原有的CSS样式不变，确保UI一致性

这种架构确保了：
- 数据更新不会影响页面布局
- 保持与原版完全一致的视觉效果
- 提供更好的可维护性和扩展性
